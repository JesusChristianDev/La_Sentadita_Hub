# Bloque B01 — Auth + Profiles + Roles + Multi-restaurante

## Objetivo
- Permitir login seguro y tener un perfil (rol + restaurante) como base de todos los permisos.

## User stories (máx 5)
- US1: Como usuario, puedo iniciar sesión y ver mi perfil (rol, restaurante).
- US2: Como usuario no logueado, si entro a /app me redirige a /login.
- US3: Como admin/office, puedo seleccionar un restaurante activo como contexto.
- US4: Como employee/manager, quedo ligado a 1 restaurante y no puedo cambiar contexto.
- US5: Como office, no veré entradas/rutas de horarios (se aplicará también en RLS cuando exista schedule).

## Entidades y permisos
- Tablas:
  - restaurants
  - profiles (PK = auth.users.id)
- RLS:
  - profiles: el usuario ve su fila; admin/office pueden ver todas.
  - restaurants: select para autenticados; escritura solo admin/office (mínimo).

## API / Use cases
- Crear cliente Supabase SSR (server/client) con cookies.
- getCurrentUserProfile(): devuelve (user + profile) o null.
- setActiveRestaurantContext(): guarda restaurant_id activo (solo admin/office).

## UI mínima
- /login (email+password)
- /app (dashboard mínimo con perfil)
- Selector restaurante visible solo para admin/office
